## Status legend

| Status | Meaning |
|---|---|
| **supported** | part of the current claims-bearing surface |
| **shadow** | retained and tested, but not claims-bearing yet |
| **experimental** | research-grade / partial / promotion pending |
| **unsupported** | should fail clearly rather than degrade silently |

## S-parameter calculation surfaces

| Primitive | Calculation API | Result schema | Evidence / status |
|---|---|---|---|
| `add_port(..., extent=None)` | `run(compute_s_params=True)` | `Result.s_params`, `Result.freqs` | E2/E3-limited on uniform Yee; M13 analytic open/short/matched/RLC extractor oracles, M11 real raw V/I replay, and M14 internal replay/passivity sweep; no broad E5 envelope |
| `add_port(..., extent=...)` | `run(compute_s_params=True)` | `Result.s_params`, `Result.freqs` | E2/E3/E4 partial for probe-fed resonance plus M12 midpoint V/I replay and M15 internal replay/passivity sweep; absolute S-matrix calibration caveated; nonuniform is shadow |
| lumped/wire `add_port(...)` | `forward(port_s11_freqs=...)` | `ForwardResult.s_params`, `ForwardResult.freqs` S11 vectors | E0/E1 uniform single-device AD contract |
| `add_msl_port(...)` | `compute_msl_s_matrix()` | `MSLSMatrixResult.S`, `.freqs`, `.Z0`, `.beta` | E5-narrow / eigenmode-blocked; thru-line + cv06b + stored-openEMS smoke + real raw 3-probe replay evidence; eigenmode/nonuniform remain blocked |
| `add_waveguide_port(...)` | `compute_waveguide_s_matrix()` | `WaveguideSMatrixResult.s_params`, `.freqs`, metadata | E5-narrow strongest current port; battery + cv11 gates only within documented rectangular-guide envelope |
| `add_coaxial_port(...)` | none promoted | none | E2-sweep/E3-gap / no-sparam: M22 analytic TEM geometry sweep plus M21 diagnostic coaxial-gap V/I replay; unsupported for high-level S-parameters |
| `add_floquet_port(...)` | none promoted | none promoted | E2/E3-modal / no-promoted-api: M18 synthetic modal oracle plus M20 real-FDTD DFT-plane replay; no RCWA/external crossval or promoted modal S-parameter calculator |
| sources, TFSF, probes, flux monitors | none | field/flux/time observables | not ports; no impedance or reference plane |

`run(compute_s_params=True)` is intentionally narrow: it populates
`Result.s_params` only for lumped/wire `add_port(...)` families. Use the
specialized calculator for MSL or waveguide ports.

## Shadow lane

Current shadow surface:

- nonuniform graded-z thin-substrate workflows
- selected smoke and convergence coverage
- current-main finite-ground-plane patch crossval evidence

Shadow means the code is real and may be tested, but the lane is not yet the
public claims-bearing baseline. For S-parameters, finite arrays or successful
pytest runs are not sufficient evidence by themselves; claims must cite an
E-level, command/script, numeric gate, artifact path, external-reference
status, and mesh/frequency/geometry envelope.

## Promotion rule

A lane or feature can be promoted to **supported** only when all of these are
present:

1. support-matrix entry
2. explicit source / observable contract where relevant
3. unit and integration tests
4. analytic, dump-replay, external-solver, benchmark, or convergence evidence
5. docs, examples, and API wording aligned to the promoted scope
