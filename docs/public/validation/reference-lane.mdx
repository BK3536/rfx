This page mirrors the canonical reference-lane contract. If it disagrees
with `docs/guides/reference_lane_contract.md` or
`docs/guides/support_matrix.md`, those guides win.

## Current reference lane

- uniform Cartesian Yee
- RF / microwave workflows
- PEC, CPML, or bounded UPML boundaries
- point/current sources
- lumped and wire ports
- specialized microstrip-line ports via `compute_msl_s_matrix()`
- waveguide ports (E5-narrow within the documented rectangular-guide envelope:
  empty-guide, PEC-short, Airy/reference-plane, dump-replay, and external
  reference artifacts; cv11 uses explicit reference-convention-aware gates)
- probes, flux monitors, Harminv resonance, benchmarked NTFF/far-field, and
  S-parameters only through the support-matrix envelopes

## Modeling rule

The reference lane solves Maxwell curl updates on a staggered Cartesian Yee
grid. Anything outside that lane must be treated as shadow or experimental
until promoted explicitly.

## Support surfaces

| Surface | Current status |
|---|---|
| materials | isotropic linear, conductive, validated dispersive subsets |
| boundaries | supported only within the matrix above |
| sources | point/current, lumped, wire, specialized MSL, waveguide |
| observables | probes, flux, Harminv, benchmarked NTFF, and S-parameters with explicit E-level envelopes |
| optimization-facing outputs | only when the proxy objective has explicit validation |

## S-parameter API boundaries

- `run(compute_s_params=True)` is the full S-matrix API for lumped/wire
  `add_port(...)` only.
- `compute_msl_s_matrix()` is the MSL-port S-matrix API.
- `compute_waveguide_s_matrix()` is the waveguide full S-matrix API.
- `forward(port_s11_freqs=...)` exposes lumped/wire S11 vectors only on the
  uniform single-device differentiable path.
- Coaxial has M17 analytic TEM formula helpers only; Floquet, source, TFSF,
  probe, and flux-monitor APIs are not promoted S-parameter calculators.
