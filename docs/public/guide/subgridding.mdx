<Aside type="caution" title="Experimental research lane">
  SBP-SAT subgridding is **not** part of the current claims-bearing public
  support surface. The retained public scope is limited to an
  **experimental all-PEC arbitrary-box lane** together with selected
  **reflector/periodic boundary subsets**, still limited to **soft point
  sources**, **point probes**, and **proxy numerical-equivalence evidence**
  against a uniform-fine reference.
</Aside>

## Current retained subset

Today’s public, documented SBP-SAT subset is intentionally narrow:

- **Boundary model** — `boundary="pec"`, all-PEC `BoundarySpec`, mixed PEC/PMC
  reflector faces, and periodic axes when the refinement box stays interior to
  that axis or spans it end-to-end; mixed PMC+periodic remains unsupported
- **Refinement geometry** — one axis-aligned all-PEC refinement box only
- **Excitation** — soft point sources only
- **Observables** — point probes only
- **Evidence level** — proxy benchmark only; true physical reflection /
  transmission validation is still deferred

Everything outside that surface should be treated as unsupported or deferred,
not as an implied near-term feature.

## Supported API shape

The current supported entry point is:

```python
from rfx import Simulation

sim = Simulation(
    freq_max=5e9,
    domain=(0.04, 0.04, 0.04),
    boundary="pec",
    dx=2e-3,
)
sim.add_refinement(
    x_range=(0.010, 0.028),
    y_range=(0.010, 0.028),
    z_range=(0.012, 0.028),
    ratio=2,
    tau=0.5,
)
sim.add_source(position=(0.016, 0.020, 0.020), component="ez")
sim.add_probe(position=(0.022, 0.020, 0.020), component="ez")
result = sim.run(n_steps=200)
```

This is a **research-grade API note**, not a promotion to supported arbitrary
local refinement.

## Hard-fail boundaries

The current SBP-SAT lane is expected to **hard-fail** rather than degrade
silently for unsupported combinations, including:

- CPML / UPML boundaries
- any absorbing `BoundarySpec` face
- periodic axes touched on only one side by the refinement box
- geometry-driven `xy_margin` auto-box refinement
- DFT planes, flux monitors, and NTFF
- TFSF excitation
- impedance point ports, wire/extent ports, coaxial ports
- waveguide ports, Floquet ports, and lumped RLC

See the public [Support Boundaries](/rfx/api/support-boundaries/) page for the
derived summary, and `docs/guides/support_matrix.md` in the repo for the
canonical contract.

## Evidence level

The current executable evidence is a **proxy benchmark**, not a calibrated
physical R/T or S-parameter benchmark:

- test file: `tests/test_subgrid_crossval.py`
- comparison: subgridded run vs uniform-fine reference
- metric: single-probe DFT amplitude / phase error
- current tolerance: amplitude error `<= 5%`, phase error `<= 5°`
- additional internal arbitrary-box proxy fixtures live in
  `tests/test_sbp_sat_box_crossval.py`

That evidence is enough to keep the lane visible as experimental research, but
not enough to promote it into the claims-bearing public surface.

## Migration note

If you need a practical thin-substrate workflow today, start with
[Non-Uniform Mesh](../nonuniform-mesh/), which is the preserved shadow lane for
that job shape.

If you need physical reflection / transmission, open-boundary behavior,
boundary coexistence, or advanced refined-region ports/observables, treat
SBP-SAT subgridding as a future roadmap item rather than a current public
feature.

## Roadmap note

The next public-doc-safe promotion step is **not** “more examples.” It is
stronger evidence: a true reflection / transmission benchmark plus the boundary
and observable contracts required to make that benchmark meaningful.
